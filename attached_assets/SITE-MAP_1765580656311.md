# Katalyst Website - Complete Site Map & User Flow

## Page Structure & Routing

### 1. Landing Page (`/`)
**File:** `01-landing-page.jsx`

**Purpose:** Primary entry point for all users. Explains value proposition and drives users to browse opportunities or sign up.

**Key Sections:**
- Hero with main value proposition
- "How It Works" - 3-tier progressive depth explanation
- Value props (Validated insights, Geographic intelligence)
- CTA to browse opportunities
- Pricing preview
- Footer with navigation

**Primary CTAs:**
- "Browse Opportunities" → `/discover`
- "Get Started" → `/signup`
- "Watch Demo" → Modal or video
- "Submit Your Problem" → `/submit`

**User Flows Starting Here:**
- **Browse Flow:** Click "Browse Opportunities" → Discovery Feed
- **Direct Search:** Use search (if implemented) → Search results → Discovery Feed
- **Sign Up Flow:** Click "Get Started" → Registration flow

---

### 2. Discovery Feed (`/discover`)
**File:** `02-discovery-feed.jsx`

**Purpose:** Browse and filter validated opportunities. Main discovery interface for organic users.

**Key Features:**
- Search bar (filters opportunities by keyword)
- Category filter dropdown
- Region filter dropdown  
- Sort options (trending, validation score, market size, growth, submissions)
- Grid of opportunity cards (2 columns)
- Load more pagination

**Opportunity Card Contents:**
- Category tag
- Trending badge (if applicable)
- Title
- Validation score (0-100)
- Description snippet
- 4 metrics: Submissions, Market Size, Growth, Region
- Top 2 pain points preview
- "View full analysis" CTA

**Navigation:**
- Click any card → `/opportunity/{id}` (Problem Detail page)
- "Submit Your Problem" floating button → `/submit`

**User Flows:**
- **Browse Flow:** Scroll cards → Click interesting opportunity → Problem Detail (Tier 1)
- **Search Flow:** Use search/filters → Find specific opportunity → Problem Detail (Tier 1)
- **Filter Flow:** Apply filters → Discover niche opportunities → Problem Detail (Tier 1)

---

### 3. Problem Detail / Opportunity Page (`/opportunity/{id}`)
**File:** `katalyst-progressive-depth.jsx` (Updated with navigation)

**Purpose:** Three-tiered progressive depth system. Users start here (Tier 1 free), then upgrade to unlock deeper tiers.

#### **Tier 1: Problem Detail (FREE)**

**Visible to all users without authentication**

**Contents:**
- Geographic market selector (6 regions with quick metrics)
- Quick validation metrics (submissions, market size, urgency)
- Problem statement
- Top 3 consumer pain points with severity
- Related opportunities cluster

**Conversion Point:**
- Upgrade CTA → "Unlock Full Research Dashboard" 
- Shows what's locked: Demographics, competitive landscape, TAM/SAM/SOM
- Price: Pro tier ($99/mo)

**User Flows:**
- **Bounce:** Not interested → Back to Discovery Feed
- **Free Exploration:** Browse related opportunities → Another Problem Detail
- **Convert to Pro:** Click upgrade → Payment/signup → Tier 2 unlocked

---

#### **Tier 2: Research Dashboard (PRO - $99/mo)**

**Requires Pro subscription**

**Tab Structure (5 tabs):**

1. **Market Validation Tab**
   - Demand signals (search volume, YoY growth, social mentions)
   - Demographics breakdown (age, income, homeownership)
   - Competitive landscape (top 3 competitors with weaknesses)

2. **Geographic Distribution Tab** ⭐ NEW
   - All 6 markets side-by-side with applicability scores
   - Top cities per region
   - Regulatory complexity assessment
   - Competition density
   - 3-phase launch strategy recommendation

3. **Problem Analysis Tab**
   - Categorized pain points with frequency & severity
   - User journey pain points
   - Edge cases and constraints

4. **Opportunity Sizing Tab**
   - TAM/SAM/SOM breakdown
   - Willingness to pay indicators by tier
   - Market timing signals

5. **Solution Pathways Tab**
   - Prioritized feature requests from users
   - Common solution approaches mentioned
   - Technical feasibility indicators

**Conversion Point:**
- Upgrade CTA → "Unlock Deep Dive Analysis"
- Shows what's locked: Execution playbook, financial models, AI assistant
- Price: Business tier ($499/mo)

**User Flows:**
- **Research Mode:** Explore all tabs → Take notes → Make decision
- **Export Basic:** Download basic reports (if included in Pro)
- **Convert to Business:** Need execution details → Click upgrade → Tier 3 unlocked
- **Share/Collaborate:** Share opportunity with team (if feature exists)

---

#### **Tier 3: Deep Dive (BUSINESS - $499/mo)**

**Requires Business subscription**

**Contents:**

**Executive Summary Section:**
- AI Assistant teaser/intro card
- Investment thesis
- Key opportunities (3 bullets)
- Key risks & mitigations (3 bullets)

**6 Main Sections (Card Grid):**

1. **Geographic Market Intelligence** ⭐ NEW
   - Priority-ranked market entry sequence
   - Market entry playbooks per region
   - Regulatory compliance guides
   - Regional pricing analysis

2. **Detailed Market Intelligence**
   - Full competitive analysis with SWOT
   - Customer segmentation models
   - Pricing strategy recommendations
   - Distribution channel analysis

3. **Product Strategy**
   - Detailed feature roadmap
   - MVP scope definition
   - User stories & acceptance criteria
   - Technical architecture considerations

4. **Financial Modeling**
   - 5-year revenue projections
   - Unit economics (CAC, LTV, LTV:CAC, Payback)
   - Customer acquisition cost estimates
   - Funding requirements

5. **Execution Playbook**
   - 90-day launch plan (3 phases with specific actions)
   - Success metrics & KPIs
   - Team composition needs
   - Risk mitigation strategies

6. **Raw Data & Exports**
   - Full consumer verbatims (all submissions)
   - Statistical analysis
   - Export options: PDF, CSV, API access
   - Methodology notes

**AI Research Assistant (Floating):**
- Floating button bottom-right: "Ask AI Assistant"
- Opens chat panel (480px × 600px)
- Pre-loaded suggested questions:
  - "What are the biggest risks with this business model?"
  - "Where should I launch this geographically?" ⭐ NEW
  - "How would you recommend pricing the product?"
  - "What should the MVP feature set look like?"
- Intelligent responses based on research data
- Can answer custom questions about the opportunity

**User Flows:**
- **Strategic Analysis:** Read executive summary → Explore relevant sections
- **Deep Research:** Use AI assistant to explore specific questions
- **Geographic Planning:** Review geographic intelligence → Plan launch strategy
- **Export Everything:** Download complete reports, data, playbooks
- **Team Collaboration:** Share with investors/co-founders
- **Execution Ready:** Move from research to building

## User Flow Summary

### Browse Flow (Organic Discovery)
```
Landing Page → Browse CTA → Discovery Feed → 
Filter/Search → Click Card → Problem Detail (Free) → 
Upgrade to Pro → Research Dashboard → 
Upgrade to Business → Deep Dive + AI Assistant
```

### Search Flow (Intent-Driven)
```
Landing Page → Search Bar → Discovery Feed (Filtered) → 
Click Relevant Result → Problem Detail (Free) → 
[Same as Browse Flow]
```

### Submission Flow (Problem Submitter)
```
Landing Page → Submit Problem → Confirmation → 
[Weeks/Months Later] → Email Notification → 
Problem Detail (Their Problem, Free) → 
Higher Conversion → Research Dashboard → Deep Dive
```

### Direct Entry Flow (Shared Link)
```
Shared Link → Problem Detail (Free) → 
Creates Account → Upgrades to see more
```

## Technical Implementation Notes

### Routing Structure (React Router example)
```javascript
<Routes>
  <Route path="/" element={<LandingPage />} />
  <Route path="/discover" element={<DiscoveryFeed />} />
  <Route path="/opportunity/:id" element={<OpportunityDetail />} />
  <Route path="/submit" element={<ProblemSubmission />} />
  <Route path="/pricing" element={<Pricing />} />
  <Route path="/signup" element={<SignUp />} />
  <Route path="/signin" element={<SignIn />} />
  <Route path="/account" element={<Account />} />
</Routes>
```

### Authentication Guards
- Free tier: No auth required for browsing
- Pro tier: Requires authenticated user with Pro subscription
- Business tier: Requires authenticated user with Business subscription
- Tier checks happen on `/opportunity/:id` page based on which tier is being viewed

### Data Loading
- **Discovery Feed:** Load all opportunities (or paginated)
- **Problem Detail:** Load specific opportunity by ID
- **Tier 2/3:** Check user subscription, load additional data if authorized
